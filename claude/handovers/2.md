3. **`SL-8.e`** — single-line override / add-product form (shop label |
   link | EAN | price | ETA | save). Build as a reusable component;
   `SL-8.d` reuses it.
4. **`SL-8.d`** — alternative items + selectable configurations.
   Schema change: `items.json` gets optional `alternatives[]` per item;
   new top-level `configurations[]` (named bundles, selectable at the
   top of the page). Sketch the schema with the user before coding.
   Bump `cnc-shopping-state-v1`.

   sparkline data is already in `prices.json.observations`; the
   provenance label depends on the `SL-9.e` `technique` sub-field, so
   add that field (default `"manual"`) before rendering.

   - `SL-8.g`: when multiple bots/techniques write divergent values for

     found" from "product found, price missing"; the latter writes a

   each annotated with a suggested cooldown to avoid rate-limiting).
   **Design discussion required first** — static MkDocs cannot write
