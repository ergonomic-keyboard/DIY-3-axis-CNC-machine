# Handover 1 — Shopping-links feature (post-MVP)

## What's been shipped (branch `include-shopping`, last commit `3149572`)

The objectives in `claude/objectives/shopping_links.md` (SL-0 → SL-9) are
implemented end-to-end. Read that file first — it is still the source of truth.

- **Dedicated Shopping page** at `docs/shopping.md`, rendered client-side
  from three static JSON files in `docs/data/`:
  - `shops.json` — countries + per-shop shipping cost / free-shipping
    threshold / currency / default ETA.
  - `items.json` — full BOM index (code → category, name, qty, image).
  - `prices.json` — append-only `observations` per `(item_code, shop)` pair.
- **Image per item** (SL-8.a) — synced from `BILLOFMATERIAL.md` by
  `tools/sync_bom_images.py`; rendered as a 3-rem thumb with a dashed
  fallback when an image is missing.
- **Refresh tooling** (SL-9):
  - `tools/refresh_prices.py` — stdlib-only, parses Schema.org `Product`
    JSON-LD first and Open Graph / microdata as fallback. Skips listing /
    search / category URLs by URL shape (no fetch attempt). Appends only
    when an observation actually changed.
  - `.github/workflows/refresh-prices.yml` — weekly + manual trigger; opens
    a PR via `peter-evans/create-pull-request@v6` so a human reviews the
    diff before publishing.
- **Reliability** documented in `developers.md`: TinyTronics / 123-3D
  parse cleanly; Conrad sits behind Cloudflare; Bol.com / Amazon.nl / AliExpress
  are in `SKIP_SHOPS` (need affiliate APIs, not scraping).

`prices.json` ships with **only verified-real entries** (5 × TinyTronics +
2 × 3D-print quote-only). Earlier PLACEHOLDER rows with made-up numbers
have been removed — the dataset is now a small honest seed, not a fake
catalogue.

## Repo orientation (skim if you forgot)

- Static MkDocs + Material site, deployed on push to `main` via
  `.github/workflows/ci.yml`. **The new feature branch does not
  auto-deploy** — merge to `main` to publish.
- Local dev: `pip install mkdocs-material && NO_MKDOCS_2_WARNING=1 mkdocs serve`.
  The dev server serves under the GitHub Pages prefix
  (`http://127.0.0.1:8000/DIY-CNC-machine/shopping/`). The shopping JS
  resolves data + image URLs from `window.location.pathname`, so this
  works both locally and in production.
- Client-side state lives in `localStorage` under `cnc-shopping-state-v1`.
  Toggle pattern was copied from `docs/assets/javascripts/toc-toggle.js`.
- `.gitignore` now excludes `site/` (build output) and `__pycache__/`.

## What was *intentionally* not built

- **Optimiser** ("suggest cheapest mix of shops"). SL-6.c says out of scope
  for v1. UI is structured so it can be added as a single extra button next
  to the sum view.
- **Affiliate APIs** for Bol.com / Amazon.nl / AliExpress. These need a
  registered account, credentials in GitHub Secrets, and a different
  fetcher per provider. Listed in `SKIP_SHOPS` so the refresh job runs
  clean instead of failing on bot walls.
- **Browser-side scraping**. Static site + CORS = not a viable path.
- **Real prices for every BOM item**. Filling concrete product URLs and
  prices for ~130 items is data work, not engineering. The schema and
  tooling are ready; entries get added as the user finds product pages.

## Open items for the next chat

Triaged previously; all of these are scoped for a fresh chat because they
need either a browser, external accounts, or a larger code change:

1. **Visual review on real device.** Image sizing on mobile, sticky summary
   on narrow viewports, country switch flow, override-drawer ergonomics,
   keyboard navigation. The current implementation was build-verified, not
   browser-verified.
2. **Bol.com / Amazon.nl / AliExpress** via affiliate APIs (PA-API,
   Plaza, Open Platform). Adds secrets management + per-provider fetcher.
3. **Scraping `og:image` from shops** to display next to the BOM image, so
   SL-8.a verification becomes two-sided ("our picture vs theirs"). Adds
   another scraping target with its own reliability story.
4. **Filling the catalogue** with real product URLs for items beyond the
   5 seeded ones. Repeatable manual / scripted work — not a code task.
5. **Optimiser button** (SL-6.c) once the dataset is dense enough that
   sum-of-cheapest leaves real money on the table.
