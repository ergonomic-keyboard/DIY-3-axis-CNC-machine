### Unresolved spec issues for the user

1. **Duplicate SL-8.a** on lines 33 and 39 of the objectives. Effectively
   the same requirement; safe to leave but worth a one-line tidy.
2. **Two SL-9.a definitions** (lines 35 and 55) that pull in opposite
   directions — one is a scheduled-PR refresher, the other is an in-page
   per-bot trigger button. The next chat must ask the user which one wins
   (likely: both, but renamed — call the new one SL-9.d).
3. **"Bot" concept (SL-8.f/g, SL-9.a-new)** is not defined. Today there
   is only one fetcher (`tools/refresh_prices.py`). Multi-bot implies a
   plugin layout — one fetcher per shop or per technique (JSON-LD, OG,
   affiliate-API). Spec needs to name expected bots before implementing.

## Repo orientation

- Branch `include-shopping`, last commit `e0e4398` (Optimist Prime).
  Diverges from `main`; not auto-deployed (CI deploys only on push to
  `main` / `master`).
- Local dev: `pip install mkdocs-material && NO_MKDOCS_2_WARNING=1 mkdocs serve`.
  Page is at `http://127.0.0.1:8000/DIY-CNC-machine/shopping/`.
- Data: `docs/data/shops.json`, `docs/data/items.json`, `docs/data/prices.json`.
  Image sync: `python3 tools/sync_bom_images.py`. Refresh: `python3 tools/refresh_prices.py [--dry-run]`.
- Client-side state lives in `localStorage` under `cnc-shopping-state-v1`.
  When the schema in shopping.js changes meaningfully (e.g. adding
  alternative-groups in SL-8.d), bump that key to invalidate old state
  cleanly.
- `.gitignore` excludes `site/` and `__pycache__/`.
