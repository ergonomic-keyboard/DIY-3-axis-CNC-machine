## SL-9.d open design question

The brief said "design discussion first". I proposed in chat:

- **Architecture**: new optional `tools/refresh_server.py` (stdlib `http.server`,
  no deps) that the user runs alongside `mkdocs serve`. Exposes
  `GET /api/health` and `POST /api/refresh?bot=<shop>-bot`. The page
  polls health once to decide whether buttons run live or render in a
  degraded "helper offline — start `python3 tools/refresh_server.py`"
  state. `mkdocs serve` auto-reloads when the helper writes
  `prices.json`.
- **Cooldown**: per-bot min interval declared in code, enforced
  server-side with HTTP 429 + `Retry-After`. Page reads `Retry-After`
  and disables the button with a countdown.
- **Toolbar placement**: third row inside the existing `.shopping-header`,
  one button per shop in scope, compact, with last-run age + cooldown
  countdown.
- **GHA SL-9.a stays** as the scheduled/unattended path.

## Schema changes the next chat must know about

### `docs/data/prices.json` — `$schema_version: 2`

Every observation carries a `technique` sub-field. Optional
`partial: true` flag marks observations that the bot saw but couldn't
price (SL-8.h). Example:

```jsonc
{
  "item_code": "E09",
  "shop": "tinytronics",
  "url": "https://www.tinytronics.nl/...",
  "ean": null,
  "observations": [
    {
      "ts": "2026-06-25T12:42:59Z",
      "price": 25.00,
      "currency": "EUR",
      "in_stock": false,
      "eta_days": 1,
      "technique": "jsonld"            // one of: jsonld | opengraph | affiliate-api | manual
    },
    {
      "ts": "2026-06-25T13:00:00Z",
      "price": null,
      "currency": "EUR",
      "in_stock": null,
      "eta_days": 1,
      "technique": "jsonld",
      "partial": true                  // SL-8.h: product found but no price
    }
  ]
}
```

## Architecture notes from this chat

- **`STATE_KEY`** is `'cnc-shopping-state-v5'`. Add the next key to the
  front of `LEGACY_STATE_KEYS` if you bump again.
- **User-added alternative codes** use `<parent>-user-<4hex>` via
  `generateUserAltCode()` (crypto.getRandomValues, retries on collision).
- **Synthetic user shops** get id `__user_<altCode>` and bypass
  country-scope filtering in `entriesForItem` so they always show
  regardless of the user's country pick.
- **snap-to-saved** lives in `snapActiveConfigFromCustom()` — called
  any time `customUses` mutates. Empty customUses snaps to `default`.
  `setEffectiveAlternative` seeds customUses from the current
  activeUses before mutating, so swaps compose on top of a saved
  configuration instead of dropping it.
- **Bot id** is derived in `botIdFor(entry, obs)`, returning null for
  manual observations and user-added shops. No `bot` field is stored
  on observations.
- **Disagreement detection** in `detectTechniqueDisagreement` keeps
  the latest observation per technique (sortedAsc iteration → latest
  wins) and reports when ≥2 techniques have different prices.
- **Partial observations** render in the popover with a dashed-red
  left-stripe and "Price missing" placeholder; the sparkline icon
  falls back to a `⋯` glyph when there's only one real price point.
- **Focus visibility** lives in a single `.shopping-app`-scoped
  `:focus-visible` rule near the top of `shopping.css`. `label:has(input:focus-visible)`
  highlights the wrapping label so users see what they're about to toggle.
