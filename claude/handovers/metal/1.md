# Handover 1 — metal plates pipeline established; next part: engine_holder_vertical_p1of2

## What changed since `0.md`

The metal-plates workflow is no longer a per-part bespoke script. It's now
a **5-stage, example-folder-driven pipeline** with three reusable scripts
under `hardware_mods/metal_plates/`. The README has been trimmed to just
the commands; everything else is in `--help` and on-screen banners.

| Script | Stages | What it does |
| --- | --- | --- |
| `rectify.py` | 2 → 3 → 5 | Reads an STL + a raw photo; auto-picks 4 widest-spread Y-holes; renders a labelled plastic-plate reference; shows it side-by-side with the photo for the 4 calibration clicks; writes the rectified PNG + metadata to `2_flattened_image/`; runs an optional measurement loop into `3_measurements/`; chains straight into the polygon trace, output in `4_outline/`. Cached calibration choice + clicks persist between runs. |
| `build_model.py` | 6 → 8 | Reads stage-4 polygon + stage-2 calibration + cached STL holes; extrudes via build123d, punches Y-axis holes that fall inside the polygon, exports STEP + STL to `5_models_and_renders/`, renders a clean plan view AND a transparent overlay of the plastic STL silhouette (procedure step 8), and writes a hole-diff JSON (step 7). `--view` opens the STEP in `f3d`/`FreeCAD` if on `$PATH`. |
| `align_outline.py` | 5b | Interactive editor on top of the overlay. Click an edge; H/V/N constraint; `1`/`2` snap to an auto-suggested parallel plastic-boundary segment; `F` cycles candidates; `M` enters a manual click-on-plastic snap mode; `G` is the group editor (enter → click edges to toggle → `G` commits, longest member becomes the anchor, all members spring to its position; afterwards nudging any member moves the whole group); `←→↑↓` nudge (Shift = fine 0.1 mm); `U` 1-step undo; `R` reset; `S` save; `D` save+exit; `Q` discard. Writes `4_outline/<name>_polygon_edited.json`; `build_model.py` prefers the edited file over the raw trace automatically. |

Matplotlib's default keymap is wiped (except `quit`) in all three scripts
so single-letter bindings (`S`, `R`, `G`, …) don't double-fire matplotlib's
native save-dialog / home-view / grid / pan / zoom actions.

Stage folder convention per example (auto-created on first `rectify` run):

```
<example>/
  0_raw_screenshots/        raw video frames (drop multiple views here)
  1_original_plastic_images/ optional reference photos of the plastic prototype
  2_flattened_image/        rectify outputs: *_rect.png, *_rect.json,
                              plate_reference.png, holes_from_stl.json
  3_measurements/           optional manual sanity-check measurements
  4_outline/                trace_polygon outputs + align_outline edited JSON
  5_models_and_renders/     STEP, STL, plan PNG, plastic overlay, hole_diff
```

## Pipeline commands for this part

```sh
EX=hardware_mods/metal_plates/examples/mid_vertical_movement/engine_holder_vertical_p1of2
STL=docs/stl_files/router/Z_MOTOR_MOUNT.stl   # confirm this matches the photos!

# 1. Move screenshots into stage 0, then flatten + trace.
mkdir -p "$EX/0_raw_screenshots"
mv "$EX/Screenshot From 2026-06-27 "*.png "$EX/0_raw_screenshots/"

nix-shell hardware_mods/metal_plates/shell.nix --run \
  "python hardware_mods/metal_plates/rectify.py --example $EX --stl $STL"
# (will prompt to disambiguate which of the 6 raw screenshots; pass --photo
#  the first time, then re-runs cache the choice)

# 2. Build the model + overlay.
nix-shell hardware_mods/metal_plates/shell.nix --run \
  "python hardware_mods/metal_plates/build_model.py --example $EX"

# 3. Tighten the polygon against the overlay (loop until satisfied).
nix-shell hardware_mods/metal_plates/shell.nix --run \
  "python hardware_mods/metal_plates/align_outline.py --example $EX"
```
