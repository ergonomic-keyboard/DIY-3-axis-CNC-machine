# Handover 0 — metal plates port

## Mission

Port the rest of the plastic STL parts to **build123d** Python models of
their aluminium equivalents, matching the build-video screenshots.
The first part (left side plate) is done and is the working template.

Source brief: `claude/objectives/metal_plates.md` —
> "Use build123d python to generate models of the metal plates. We start
> with the side plate." Metal plates are 10 mm or 6 mm aluminium and
> "(generally) have the same holes but different shapes" than the plastic
> versions.

## What's done

Working example in `hardware_mods/metal_plates/`:

| File | Role |
|---|---|
| `shell.nix` | `nix-shell` recipe wiring `libGL`, `expat`, `fontconfig`, `freetype`, `libxext`, `libx11`, `libxrender`, `libice`, `libsm`, `libxi`, `libxmu`, `stdenv.cc.cc.lib` into `LD_LIBRARY_PATH` so OCP (OpenCascade) can load on NixOS. Also auto-activates `.venv` if present. |
| `extract_holes.py` | Parses a binary STL; walks every triangle-pair edge parallel to X / Y / Z; per axis-and-length bucket greedily peels circles with `KDTree` + `sklearn` algebraic fit; writes `holes.json`. **Robust across hole sizes** (1.5 mm to 25 mm) because clustering is per-length, not per-distance. |
| `inspect_stl.py` | One-shot dumper: bbox + dominant-normal histogram + per-axis face Y-plane modes. Use for any new STL before extracting. |
| `scan_upper.py` | Diagnostic for parts where features sit on perpendicular fins — counts X / Y / Z-parallel seams per region so you can confirm which axis the cylindrical features use. |
| `side_plate_left.py` | The build123d model itself. Polyline silhouette in the X-Z plane, extruded along +Y by `PLATE_THICKNESS`, holes subtracted in-sketch. Exports STEP + STL. |
| `render_plan.py` | matplotlib top-view PNG of the silhouette + holes for sanity-checking. |
| `holes.json`, `side_plate_left_metal.{step,stl}`, `side_plate_left_metal_plan.png` | Generated outputs (committable, but also re-generatable). |

Run: `nix-shell hardware_mods/metal_plates/shell.nix --run "python hardware_mods/metal_plates/side_plate_left.py"`

The venv lives at the **repo root** `.venv/` and is created with
`python3 -m venv .venv --system-site-packages`, then
`pip install build123d numpy-stl matplotlib scikit-learn`. The `shell.nix`
sources it automatically.

For each part:
1. **Survey** the STL with `inspect_stl.py` to confirm orientation
   (the left plate had Y as the thickness axis — others may differ).
2. **Extract holes** by adapting `extract_holes.py` to point at the new STL.
3. **Read the screenshot** for the metal silhouette — trace it as an
   `OUTLINE` polygon. Aim for the shape to *enclose* every detected hole
   with a few mm of clearance.
4. **Copy `side_plate_left.py`** to e.g. `side_plate_right.py`, swap STL
   path, `OUTLINE`, and any hole-filtering rules.
5. **Render** with `render_plan.py` (parameterise it on a part-name arg)
   to verify holes land inside the silhouette before claiming success.

## Gotchas already paid for

- **NixOS + OCP** needs `libexpat.so.1` and a bag of X11 libs on
  `LD_LIBRARY_PATH`. `shell.nix` already does it — don't recreate the
  venv outside `nix-shell` or imports break with "libGL.so.1 not found".
- The plastic side plate's **motor-mount hole is on a perpendicular fin**
  (its cylinder axis runs along **X**, not Y). On the flat metal plate
  this projects to (fin-midplane-X, Z). The extractor handles all three
  axes; the filter in `metal_holes()` keeps only the X-axis hole with
  `cz > 100` to ignore false positives from stepped pockets.
- The STL has **stepped counterbores**: the same (cx, cz) point shows up
  twice — once at the screw-bore diameter and once at the head pocket
  diameter. `metal_holes()` keeps the **smallest** diameter per (cx, cz)
  for a clean through-hole. If the user later wants the counterbores
  visible on the metal plate too, swap to keeping both.
- `Polyline` is a **BuildLine** primitive, not a BuildSketch one. Wrap
  it inside `with BuildLine():` and then call `make_face()`.
- The greedy peel uses `rms_tol=0.05` (5% of radius). Loosen to `0.08`
  if a part has tessellation noise that rejects valid holes; tighten
  if false positives leak in.
